# HEAXHub 빌드 보강용 pip 패키지 목록 (오프라인 폴백 미러의 source of truth).
#
# 용도.
#   온라인 빌드 호스트에서 deploy/apptainer/mirror-to-drive.sh 가 이 파일을 읽어
#   pip download 로 wheel/sdist 를 받아 Google Drive(pkgs-<TS>/ + latest/)에 올린다.
#   서버에서는 deploy/apptainer/mirror-from-drive.sh 가 그걸 받아
#   var/pkg-mirror/pip/ 에 풀어 둔다 (Docker Hub/PyPI 막힌 망에서 폴백 미러로 사용).
#
# 형식.
#   각 행에 패키지 하나. requirements.txt 와 동일 문법.
#     package==1.2.3      # 정확한 버전 고정(재현성 권장).
#     package>=1.2,<2     # 범위 지정.
#     package             # 버전 미고정(최신).
#   '#' 로 시작하는 행과 빈 행은 무시된다.
#
# 주의.
#   여기 적힌 패키지는 backend/requirements.txt 와 별개인 "빌드/런타임 보강" 목록이다.
#   실제 앱 의존성은 backend/requirements.txt 에서 관리하고,
#   이 파일에는 그 외 추가로 미러링이 필요한 패키지만 넣는다.
#
# 예시(주석 처리됨 — 필요 시 아래 형태로 주석을 풀어 사용).
#   wheel==0.43.0
#   setuptools>=69
#   pip>=24.0
